Adversarial Attacks

Exploiting the Vulnerability of Neural Networks

01

What Are Adversarial Attacks?

Adversarial attacks are carefully crafted input perturbations designed to fool neural networks into making incorrect predictions. These perturbations are typically imperceptible to humans but cause catastrophic failures in deep learning models.

Key Insight: Neural networks learn highly non-linear decision boundaries in high-dimensional spaces. Adversarial examples exploit the geometry of these boundaries — tiny steps in the right direction can cross decision boundaries while remaining visually identical to humans.

First demonstrated by Szegedy et al. (2013), adversarial examples pose serious security concerns for real-world AI deployments in autonomous vehicles, medical imaging, and security systems.

The Geometry Problem

In high-dimensional input spaces, most volume lies near the boundary of the input manifold. Neural networks trained via ERM generalize well on the data manifold but leave vast regions of input space unconstrained.

Adversarial Direction Exists Because: dim(input) ≫ dim(manifold)
∴ ∃δ: ‖δ‖∞ ≤ ε, f(x+δ) ≠ f(x)

The input dimensionality (e.g. 224×224×3 = 150,528) vastly exceeds the intrinsic dimensionality of natural images, leaving exploitable directions.

Taxonomy of Adversarial Attacks

Adversarial attacks are categorized along multiple dimensions: adversary knowledge, perturbation objective, timing relative to training, and the attack surface targeted.

White-Box Attacks

The attacker has complete knowledge of the model: architecture, weights, training data, and gradients. This is the strongest attack assumption.

CAPABILITIES
  • ✓ Direct gradient computation
  • ✓ Loss landscape exploration
  • ✓ Architecture exploitation
  • ✓ Iterative refinement
Examples: FGSM, PGD, C&W, DeepFool
Black-Box Attacks

The attacker has no direct access to model internals. Must rely on query outputs or transferability of adversarial examples.

STRATEGIES
  • ▸ Transfer-based: craft on surrogate model
  • ▸ Score-based: use output probabilities
  • ▸ Decision-based: use hard labels only
  • ▸ Query-efficient optimization
Examples: ZOO, HopSkipJump, NES Attack
Untargeted Attacks

Goal: cause any misclassification. Maximize the loss for the true class, pushing the sample away from the correct decision boundary in any direction.

Objective: max_δ J(f(x+δ), y_true)
subject to ‖δ‖ ≤ ε

Easier to achieve; any wrong label is a success. Useful for denial-of-service attacks on classifiers.

Targeted Attacks

Goal: cause the model to predict a specific wrong class y_target. More constrained and harder to achieve, but more dangerous.

Objective: min_δ J(f(x+δ), y_target)
subject to ‖δ‖ ≤ ε

Critical for high-stakes: make a stop sign look like a speed limit sign at a specific value.

INFERENCE-TIME

Evasion Attacks

Attack occurs after training is complete. The model weights are fixed; only the input is manipulated to evade detection or change prediction. Most common attack scenario.

STAGE: Deployment → Attack → Wrong Output
TRAINING-TIME

Poisoning Attacks

Malicious data is injected into the training set before or during training. The resulting model has a backdoor or degraded performance on targeted inputs.

STAGE: Data Collection → Poison → Train → Deploy
BACKDOOR

Trojan Attacks

A special case of poisoning where the model behaves normally on clean inputs but outputs a target class when a specific trigger pattern (e.g., a pixel patch) is present.

TRIGGER: Small patch → Always predict: "TARGET"
PHYSICAL

Physical-World Attacks

Adversarial perturbations applied in the physical domain — printed patches, adversarial glasses, stop sign stickers — that survive camera capture and preprocessing.

EXAMPLE: Adversarial patch on shirt evades person detection
QUERY-FREE

Transfer Attacks

Adversarial examples crafted on a surrogate model often transfer to other models. Exploits the observation that different models learn similar decision boundaries.

Surrogate → Adv. Example → Target Model
MODEL

Model Extraction

Query the target model to reconstruct a functional copy. The extracted model can then be used as a white-box surrogate for crafting more effective adversarial examples.

Queries + Labels → Surrogate Clone

03

Mathematical Foundations & Attack Methods

Each attack method defines a specific optimization problem over the perturbation δ, constrained by a norm ball. The key insight is that neural network loss surfaces have locally linear behavior that can be exploited.

FGSM — Fast Gradient Sign Method

Proposed by Goodfellow et al. (2015). Exploits the linearity hypothesis: the loss function is approximately linear in the input, so the steepest increase in loss is along the sign of the gradient.

A single gradient step in the sign direction. Fast, but suboptimal. The sign operation ensures every pixel moves by exactly ε in the worst direction.

Strength: Computationally trivial (one forward + backward pass)
Weakness: Single step; easily defended by robust training

FGSM Update Rule: x_adv = x + ε · sign(∇ₓ J(θ, x, y))
Gradient Computation: ∇ₓ J = ∂L(f(x;θ), y) / ∂x
PARAMETERS
ε = perturbation budget (L∞)
J = cross-entropy loss
θ = fixed model weights
sign() = sign of each gradient element
PGD — Projected Gradient Descent

Madry et al. (2018) extended FGSM to multiple iterative steps, projecting back onto the L∞ ball after each step. Considered the strongest first-order attack.

PGD is essentially repeated FGSM with a smaller step size α, with random initialization within the ε-ball. The projection Π ensures we stay within budget.

Strength: Finds near-optimal adversarial examples; used as gold-standard evaluation
Weakness: Expensive (K forward/backward passes)

PGD Iteration (k steps): x⁰ = x + Uniform(-ε, ε)

xᵗ⁺¹ = Πε(xᵗ + α · sign(∇ₓ J(θ, xᵗ, y)))
Projection onto L∞ Ball: Πε(x') = clip(x', x-ε, x+ε)
C&W Attack — Carlini & Wagner (2017)

Formulates adversarial example generation as an optimization problem with a custom objective that avoids the cross-entropy pitfall of saturation near zero loss.

Uses a change of variable w = tanh⁻¹(2x - 1) to naturally enforce [0,1] pixel bounds, avoiding explicit box constraints. Iterates with Adam optimizer.

Strength: Finds minimum-norm adversarial examples; breaks many defenses
Weakness: Very slow; requires many optimizer iterations

C&W Objective (L2): min_δ ‖δ‖₂² + c · f(x + δ)
Adversarial Objective f: f(x') = max(Z(x')[y_true] - max_{j≠y} Z(x')[j], -κ)
Z(x') = logit outputs | κ = confidence margin | c = tradeoff constant
DeepFool — Minimum-Norm Perturbation

Moosavi-Dezfooli et al. (2016). Finds the minimum L₂ perturbation needed to cross the decision boundary by iteratively linearizing the classifier and computing the exact distance to the nearest hyperplane boundary.

Strength: Near-minimal perturbation; great for measuring model robustness
Weakness: Iterative; may not converge for highly nonlinear models

Affine Approximation at xᵗ: Δxᵗ = - f_k(xᵗ)/‖∇f_k(xᵗ)‖₂² · ∇f_k(xᵗ)
Nearest Boundary Class k*: k* = argmin_{k≠ŷ} |f_k(xᵗ)| / ‖∇f_k(xᵗ)‖₂
Universal Adversarial Perturbations (UAP)

Moosavi-Dezfooli et al. (2017). A single universal perturbation that causes misclassification for most images in a dataset — input-agnostic.

The UAP exploits the geometric structure of the decision boundaries. By iterating over data points and aggregating DeepFool perturbations, a common direction is found.

UAP Objective: P(f(x + v) ≠ f(x)) ≥ 1 - δ

subject to ‖v‖ₚ ≤ ξ
v = universal perturbation | δ = desired fooling rate | ξ = perturbation budget. A single v fools any image with probability ≥ (1-δ).

Robustness Enhancement & Defense

Defending against adversarial attacks is an active arms race. Defenses fall into several categories: proactive (model hardening), reactive (input preprocessing), and detection (identify adversarial inputs).

Adversarial Training

Strongest known defense. Augment the training set with adversarial examples during training. The model learns to be robust to adversarial perturbations by solving a min-max optimization.

Madry's Adversarial Training: min_θ E_(x,y)~D [ max_{δ:‖δ‖≤ε} J(θ, x+δ, y) ]
Inner max: Find the worst-case perturbation (PGD)
Outer min: Minimize loss on worst-case inputs
Cost: 3-10× more compute; slight drop in clean accuracy
Certified Defenses

Provable robustness within an ε-ball. Instead of empirical adversarial training, these methods provide mathematical certificates that no perturbation within budget can change the prediction.

  • ▸ Randomized Smoothing: Output class is certified for all ‖δ‖₂ ≤ R
  • ▸ IBP (Interval Bound Propagation): Propagate input bounds through layers
  • ▸ Lipschitz Constraints: Bound sensitivity via spectral normalization
Randomized Smoothing Certificate: P(g(x+ε) = c) ≥ p_A
Certified radius: R = σ/2 · (Φ⁻¹(p_A) - Φ⁻¹(p_B))
Input Transformation Defenses

Apply transformations to inputs before feeding them to the model. These aim to destroy adversarial perturbations while preserving semantic content. However, many are broken by adaptive attacks that account for the transformation.

Feature Squeezing

Reduce bit-depth or apply spatial smoothing. Compare model output on original vs. squeezed input — large divergence indicates adversarial input.

JPEG Compression

High-frequency perturbations are removed by JPEG quantization. Effective against L∞-small perturbations but can hurt clean accuracy.

Input Randomization

Randomly resize and pad inputs before classification. Disrupts gradient-based attacks by introducing stochasticity in the pipeline.

Pixel Deflection

Randomly replace a subset of pixels with adjacent pixel values, then apply wavelet denoising. Destroys structured perturbation patterns.

Neural Purification

Use a denoiser (e.g., diffusion model) to project adversarial input back onto the clean data manifold before classification.

Adversarial Denoising

Train a denoising autoencoder specifically to remove adversarial perturbations. Output is a cleaned version passed to the classifier.

Architectural & Training Strategies

Distillation

Train a student model on soft labels from a teacher. Defensive distillation (Papernot 2016) reduces gradient magnitude, though largely broken by C&W attacks.

Spectral Normalization

Constrain the Lipschitz constant of each layer by normalizing the spectral norm of weight matrices. Bounds model sensitivity to input perturbations.

Ensemble Methods

Combine predictions from multiple diverse models. Adversarial examples that fool one model are less likely to fool all models simultaneously.

TRADES

Trade-off-inspired Adversarial Defense via Surrogate-loss minimization. Adds a regularization term that explicitly trades clean accuracy for robustness.

TRADES Objective: min_f E[L(f(x), y)] + β · E[ max_{‖δ‖≤ε} KL(f(x+δ) ‖ f(x)) ]

Quick Reference Notes

Condensed reference covering key formulas, definitions, attack types, and defense strategies for fast review.

Core Definitions

  • Adversarial example: input x' = x + δ where f(x') ≠ f(x) but ‖δ‖ ≤ ε
  • Perturbation budget ε: maximum allowed distortion
  • Threat model: defines adversary's knowledge, goal, and budget
  • Transferability: adversarial examples generalize across models
  • Fooling rate: % inputs misclassified under a given attack
  • Robust accuracy: accuracy under worst-case attack

Key Attack Formulas

  • FGSM: x' = x + ε·sign(∇ₓJ(θ,x,y))
  • PGD: xᵗ⁺¹ = Πε(xᵗ + α·sign(∇ₓJ)) with random init
  • C&W: min ‖δ‖₂ + c·f(x+δ), f based on logit margin
  • DeepFool: step toward nearest decision hyperplane
  • UAP: find v such that P(f(x+v)≠f(x)) ≥ 1−δ for all x
  • BIM/I-FGSM: iterative FGSM with clip after each step

Attack Taxonomy

  • White-box: full model access (arch, weights, gradients)
  • Black-box: only query access; use transfer or score-based
  • Untargeted: any wrong prediction is success
  • Targeted: force specific incorrect class
  • Evasion: attack at inference time (most common)
  • Poisoning: corrupt training data; backdoor/trojan attacks
  • Physical: adversarial patches that survive real-world capture

Defense Strategies

  • Adversarial training (Madry): min-max optimization; strongest empirical defense
  • TRADES: explicit clean vs. robust accuracy tradeoff
  • Randomized smoothing: certified L₂ robustness via Gaussian noise
  • IBP: interval bound propagation for certified robustness
  • Feature squeezing: compare squeezed vs. original output
  • Input preprocessing: JPEG, randomization, denoising
  • Spectral norm: bound Lipschitz constant of network layers

Important Papers

  • Szegedy 2013 — First adversarial examples (L-BFGS)
  • Goodfellow 2015 — FGSM + linear hypothesis (ICLR 2015)
  • Papernot 2016 — Defensive distillation (S&P 2016)
  • Madry 2018 — PGD attack + adversarial training (ICLR 2018)
  • Carlini & Wagner 2017 — C&W attack; breaks most defenses
  • Cohen 2019 — Certified robustness via randomized smoothing
  • Zhang 2019 — TRADES: clean/robust tradeoff theorem

Evaluation Checklist

  • Always use adaptive attacks (account for defense in attack)
  • Report both clean accuracy and robust accuracy
  • Evaluate under multiple norms (L∞, L₂) and ε values
  • Use PGD with enough iterations (≥20) as baseline
  • Check AutoAttack (Croce 2020) for reliable evaluation
  • Avoid "security through obscurity" — gradient masking is not defense
  • RobustBench (robustbench.github.io) for standardized comparison

Common Pitfalls

  • Gradient masking: model obfuscates gradients but isn't truly robust
  • Broken baselines: many defenses fail under adaptive/white-box attacks
  • Overfitting to specific ε: robust at ε=8 doesn't mean robust at ε=16
  • Clean accuracy vs. robust accuracy tradeoff is fundamental (not just tuning)
  • Standard PGD may not be sufficient; use AutoAttack for honest evaluation
  • Transfer doesn't always work: need sufficient surrogate model similarity

Real-World Concerns

  • Autonomous driving: stop sign → speed limit sign attacks
  • Face recognition: adversarial glasses fool commercial systems
  • Medical imaging: adversarial X-rays misdiagnosed by AI
  • NLP: adversarial text via synonym replacement or character attacks
  • Malware detection: adversarial perturbations on binary features
  • Physical adversarial patches effective in real cameras at multiple angles
